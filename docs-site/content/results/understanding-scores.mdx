# Understanding Scores

ClearSight assigns a 0-100 accessibility score to each scanned page and aggregates them at the site level.

## Page Score

Each page gets an individual score based on the number and severity of issues found. The score is generated by AI analysis considering:

- Total number of issues
- Severity distribution (critical issues weigh more heavily)
- Issue types (confirmed violations vs potential issues)

### Score Ranges

| Score | Label | Color | Meaning |
|-------|-------|-------|---------|
| 80-100 | Good | Green | Few or minor issues |
| 50-79 | Needs Work | Yellow | Significant issues to address |
| 0-49 | Poor | Red | Critical accessibility barriers |

## Site Score

The site-level score is the **average of all page scores** from the latest crawl. This gives you a holistic view of your site's accessibility.

## Score Over Time

By running multiple crawls, you can track your score trajectory. Each crawl stores its overall score, so you can see improvement (or regression) in the crawl history.

## Fallback Scoring

If AI enrichment fails, ClearSight uses a simple heuristic: start at 100, subtract 5 points per issue. This ensures you always get a score even when the LLM is unavailable.
